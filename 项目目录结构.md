# AgentFlow 项目目录结构

> 前端：Vue3 + Vite + TypeScript + Pinia + Vue Router + **Vue Flow**（画布）+ **Ant Design Vue**（UI）
>
> 后端：Python + FastAPI + LangGraph + LangChain + MySQL + Redis + 向量库

---

## 1. 总体结构

采用前后端分离的 **monorepo** 组织（也可拆成两个仓库）：

```
AgentFlow/
├── frontend/                   # Vue3 前端
├── backend/                    # Python 后端
├── docs/                       # 项目文档
│   ├── 功能开发文档.md
│   └── 项目目录结构.md
├── docker/                     # 容器编排
│   ├── docker-compose.yml      # mysql + redis + 向量库 + sandbox
│   └── sandbox/                # 代码沙箱镜像
├── .gitignore
└── README.md
```

### 前端关键模块说明

| 模块 | 作用 | 对应需求 |
|------|------|----------|
| `components/canvas/` | Vue Flow 画布、节点渲染、连线 | 可视化编排 |
| `components/canvas/config/IOSchemaEditor.vue` | 编辑节点输入/输出字段 | 节点 IO 字段规范 |
| `composables/useVariableRef.ts` | `{{node.field}}` 引用选择与类型校验 | 变量系统 |
| `components/code-editor/` | Monaco 编辑器 + 选语言 + 模板 | 自定义代码节点 |
| `components/debug/` | 节点 IO/状态/错误日志、Last Run | 工作流调试日志 |
| `components/knowledge/` | 上传、切分、索引、检索设置 | 知识库上传 + RAG |
| `views/plugin/`、`views/marketplace/` | 插件市场、工作流市场 | 两个市场 |

> **DSL 双向映射**：画布上的 Vue Flow `nodes/edges` 与后端 `Workflow DSL` 通过 `utils/dsl.ts` 互转——保存时序列化为 DSL 上传，加载时反序列化渲染到画布。这是前后端的契约边界。

### 后端分层说明

请求自上而下穿过这几层，职责清晰：

```
routes（接口）→ service（业务编排）→ repository（数据访问）→ models（ORM）
                     │
                     ├─ engine（工作流编译与运行）
                     ├─ knowledge（RAG）
                     ├─ plugins（插件）
                     └─ sandbox（代码执行）
```

| 层 | 目录 | 职责 |
|----|------|------|
| 接口层 | `api/routes/` | HTTP 路由、参数校验、SSE |
| 业务层 | `*/service.py` | 业务编排、跨模块协调 |
| 引擎层 | `engine/` | DSL 编译、StateGraph 运行、变量池 |
| 能力层 | `knowledge/`、`plugins/`、`sandbox/` | RAG、插件、代码执行 |
| 数据层 | `repositories/` + `models/` | ORM 与数据访问 |
| 存储层 | `state/`、`core/` | Redis / 向量库 / 对象存储 |

## 4. 前后端契约边界

| 契约 | 前端 | 后端 | 说明 |
|------|------|------|------|
| Workflow DSL | `types/dsl.ts` | `engine/dsl.py` | 画布 ↔ 可执行图的中间表示 |
| 节点 IO Schema | `types/node.ts` | `engine/nodes/base.py` | 字段类型、连线校验依据 |
| 节点日志 | `types/log.ts` | `schemas/log.py` | 调试面板渲染数据 |
| SSE 流式 | `utils/sse.ts` | `routes/workflow.py` | 调试运行时逐节点推送状态 |

> **关键约定**：前端 `types/` 与后端 `engine/dsl.py`、`schemas/` 必须字段对齐。DSL 是这套系统的「普通话」——前端画布说它、后端引擎也说它，二者靠它通信。

## 5. 本地启动（开发）

```bash
# 基础设施
docker compose -f docker/docker-compose.yml up -d   # mysql + redis + 向量库

# 后端
cd backend
uv sync                       # 或 pip install -r requirements.txt
alembic upgrade head          # 建表
uvicorn main:app --reload     # http://localhost:8000

# 前端
cd frontend
pnpm install
pnpm dev                      # http://localhost:5173 （代理 /api → :8000）
```
